A research-driven machine learning and UX project that translates emerging nutrition science into everyday, personalized recipe recommendations.
Main recommendation view displaying cancer-beating scores and a ranked list of recipes aligned to user preferences.

1. Introduction

Recent nutritional research, particularly the Hyperfoodsframework, shows that everyday ingredients contain molecules linked to cancer prevention. Despite this progress, most consumers still make food choices based on calories or macronutrients, with limited visibility into the long-term health impact of ingredients and recipes.
This project developed a personalized food recommendation system that ranks everyday recipes by their potential cancer-beating properties, while adapting to individual taste preferences, cooking habits, and comfort levels. The system combines scientific scoring, recommender techniques, and a clean interface to make cancer-preventive eating practical and approachable.

2. The Challenge

Existing “healthy eating” tools suffer from several limitations:

  • Binary nutrition labels:Recipes are categorized as “healthy” or “unhealthy” without deeper, ingredient-level insight.
  • Lack of personalization:Taste, culture, budget, and cooking comfort are rarely integrated into recommendations.
  • Weak connection to research:Scientific findings on cancer-beating molecules are not translated into everyday food decisions.
The goal of the system was to bridge this gap by combining cancer-related nutrition science with recommender systems and an intuitive user experience.

3. Approach & Strategy

The solution was guided by three design principles:
  1. Scientific validity:Construct a defensible recipe-level score from ingredient-level cancer-beating molecule data.
  2. Personal relevance:Reflect each user's flavor preferences, cuisines, and everyday cooking habits.
  3. Simplicity and trust:Present complex models through clear, understandable UI affordances and feedback loops.
These principles shaped both the architecture of the recommendation engine and the design of the user interface.

4. System Architecture

The system uses a hybrid recommendation approach that combines content-based filtering, collaborative filtering, and a cancer-beating molecule score (CBMS) to rank recipes. High-level architecture: content-based and collaborative filtering blocks feed into a cancer-beating molecule scoring block, which outputs a final ranked list of recipes. In the architecture, a content-based (CB) module filters recipes based on explicit preferences and ingredient attributes, while a collaborative filtering (CF) module leverages taste proximity between users. Both modules output preference scores which are then combined, filtered against user history, and enriched with the CBMS to produce a final recommendation list.

5. Personalization & Taste Modeling

Personalization relies on a structured representation of taste. The system captures:
  • Flavor intensities (e.g., spicy, sweet, savory, smoky, sour, umami)
  • Preferred cuisines (e.g., Italian, Japanese, Mediterranean)
  • Frequently used ingredients and familiar dishes
  • Explicit feedback via likes, favorites, and “not for me” actions
Taste Profile screen capturing long-term flavor preferences and preferred cuisines, used as input into the content-based recommender. These inputs are used to construct a user taste profile, which drives content-based recommendations and serves as a feature vector for collaborative filtering. This structure helps the system stay aligned with what users actually enjoy, rather than only what is nutritionally optimal.

6. Cancer-Beating Molecule Score (CBMS)

A central component of the system is the Cancer-Beating Molecule Score (CBMS), which estimates how strongly a recipe aligns with cancer-preventive nutritional patterns. Because no existing dataset directly maps recipes to such scores, a multi-step scoring framework was developed:
  1. Map ingredients in a recipe to cuisine and ingredient categories.
  2. Associate cuisines and ingredients with cancer-beating molecule data derived from Hyperfoods.
  3. Weight contributions based on ingredient prominence and quantity.
  4. Adjust for preparation methods that may reduce or preserve beneficial compounds.
  5. Normalize scores to allow fair comparison across different cuisines.
Recipe detail view surfacing the Cancer-Beating Score (e.g., 94/100) alongside ratings, exclusion toggle, and ingredient details, making the underlying nutrition model visible to users. The CBMS is independent of personal taste and is later combined with preference scores in the ranking layer, ensuring that every recommendation meets a minimum health standard.

7. Recommendation Experience

The final recommendation pipeline merges taste and health signals into a single score for each candidate recipe. The system:
  • Generates candidate recipes from content-based and collaborative filtering modules.
  • Removes recipes previously disliked or already shown frequently.
  • Combines preference scores with CBMS via a weighted formula.
  • Ranks recipes to produce a personalized, health-aligned list.
Recommendation feed showing a balance of familiar dishes and new options, each annotated with a cancer-beating score to support informed choices. This ranked list keeps users within a comfortable flavor range while nudging them toward recipes with stronger cancer-preventive profiles.

8. User Trust & Error Correction

Since the system can ingest images of meals and infer recipes, handling misclassifications is critical for trust. A dedicated correction flow allows users to confirm or change detected recipes when the model prediction is inaccurate. Recipe scanner correction dialog that lets users override incorrect detections and select the correct recipe, closing the loop between model predictions and real-world usage. This human-in-the-loop mechanism improves data quality over time and makes the system more transparent and reliable for users.

9. Outcomes

Functional Outcomes

  • Recommended recipes consistently surfaced high cancer-beating scores while remaining aligned with user taste profiles.
  • Reinforcement from ongoing ratings improved personalization as users interacted with the system.
  • The interface made complex nutritional ideas accessible through simple visual cues such as scores and tags.

Qualitative Feedback

  • Suggestions felt “close to what I usually cook, but slightly healthier.”
  • The cancer-beating score made the scientific layer feel concrete and understandable.
  • The correction flow increased confidence that the system would learn and improve over time.

10. Key Takeaways

The project illustrates how evidence-based science, recommender systems, and UX can work together to enable healthier behavior without forcing drastic lifestyle changes. Key learnings include:
  • Scoring frameworks must balance technical rigor with interpretability to be meaningful for end users.
  • Personalization is most effective when it incorporates both explicit preferences and behavior-derived patterns.
  • Human-in-the-loop mechanisms are essential in ML-driven products to maintain trust and improve data quality.
  • Gradual, “adjacent” suggestions are more impactful than radical changes when encouraging healthier habits.
